## Routing - Navigate Between Pages

Kaioken's built-in router makes it easy to create single-page applications with multiple views. Learn how to set up routes, navigate between pages, and handle dynamic parameters.

### Basic Router Setup

Wrap your app with the Router component and define routes:

```jsx
import { Router, Route } from "kaioken/router"

export function App() {
  return (
    <Router>
      <Route path="/" element={<HomePage />} />
      <Route path="/about" element={<AboutPage />} />
      <Route path="/users" element={<UsersPage />} />
    </Router>
  )
}
```

### Navigation with Link

Use the `Link` component for client-side navigation:

```jsx
import { Link } from "kaioken/router"

function Navigation() {
  return (
    <nav>
      <Link to="/">Home</Link>
      <Link to="/about">About</Link>
      <Link to="/users">Users</Link>
    </nav>
  )
}
```

### Dynamic Routes

Capture URL parameters with dynamic segments:

```jsx
// Route definition
<Route path="/users/:id" element={<UserDetailPage />} />

// Access parameters in component
function UserDetailPage() {
  const router = useRouter()
  const userId = router.params.id
  
  return <h1>User ID: {userId}</h1>
}
```

### Programmatic Navigation

Navigate programmatically using the `navigate` function:

```jsx
import { navigate, useRouter } from "kaioken/router"

function MyComponent() {
  const router = useRouter()
  
  const goToUser = (id) => {
    navigate(\`/users/\${id}\`)
  }
  
  return (
    <div>
      <p>Current path: {router.pathname}</p>
      <button onClick={() => goToUser(123)}>Go to User 123</button>
    </div>
  )
}
```

### Router Hook

The `useRouter` hook provides access to routing information:

```jsx
const router = useRouter()

// Available properties:
router.pathname  // Current path
router.params    // Route parameters
router.query     // Query string parameters
```

### Key Features

1. **Client-side Navigation**: Fast transitions without page reloads
2. **Dynamic Routes**: Capture URL parameters with `:param` syntax
3. **Programmatic Navigation**: Navigate from event handlers and effects
4. **Router State**: Access current route information in components
5. **Nested Routes**: Support for complex routing hierarchies

Try navigating between different pages in the demo. Notice how the URL changes and components update without page reloads!
