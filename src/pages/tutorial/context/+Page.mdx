## Context - Share State Across Components

Context provides a way to share state between components without passing props through every level of the component tree. This is perfect for themes, user authentication, or any global application state.

### Creating Context

First, create a context and provider:

```jsx
import { createContext, useContext } from "kaioken"

const ThemeContext = createContext()

function ThemeProvider({ children }) {
  const [theme, setTheme] = useState("light")
  
  return (
    <ThemeContext.Provider value={{ theme, setTheme }}>
      {children}
    </ThemeContext.Provider>
  )
}
```

### Using Context

Consume context in child components:

```jsx
function Header() {
  const { theme, setTheme } = useContext(ThemeContext)
  
  return (
    <header>
      <h1>Current theme: {theme}</h1>
      <button onclick={() => setTheme(theme === "light" ? "dark" : "light")}>
        Switch Theme
      </button>
    </header>
  )
}
```

### Custom Hooks

Create custom hooks to make context usage cleaner:

```jsx
function useTheme() {
  const context = useContext(ThemeContext)
  if (!context) {
    throw new Error("useTheme must be used within ThemeProvider")
  }
  return context
}
```

### Key Benefits

1. **No Prop Drilling**: Share state without passing props through intermediate components
2. **Clean API**: Custom hooks provide a clean interface
3. **Type Safety**: TypeScript support for context values
4. **Performance**: Only components using context re-render when it changes

### When to Use Context

- **Theme/UI Settings**: Colors, fonts, layout preferences
- **User Authentication**: Current user, login status
- **Language/Locale**: Internationalization settings
- **Application State**: Shopping cart, user preferences

Try changing the theme and colors in the editor to see how context shares state across all components!
